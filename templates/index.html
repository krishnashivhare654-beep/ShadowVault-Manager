Cryptographic Architecture: ShadowVault
1. AES-256 Symmetric Encryption

Utilizing Python's `cryptography` library (specifically the `cryptography.fernet` module), ShadowVault implements the global standard **AES-256 (Advanced Encryption Standard)** in symmetric key mode.

  • Plaintext Input: The password string the user enters.
  • The Key: A unique master.key file is generated locally. Fernandez handles Key Derivation automatically.
  • Symmetric Nature: The *same* key file is required for both encrypting and decrypting the data. Fernet handles IVs and Authentication Tags.
2. Local Database & Zero-Knowledge architecture

The backend is managed via Flask and **SQLAlchemy** connected to an SQLite instance (`instance/vault.db`). The architecture follows the **Zero-Knowledge** model: the applicationdeveloper (KrishnaShivare) can never know or retrieve a user's master key. Only those with physical access to the local master.key can read the ciphertext.

Quick Start Guide
  1. Access the **Vault** section via the button or navbar.
  2. Enter credential details (e.g., Google, user@gmail.com, secretpass).
  3. Click **Secure & Save** to initiate encryption.
  4. The entry will be stored in the database. The original password is lost; only the green unreadable ciphertext is retained.
  5. Your `master.key` file in the root folder is the ONLY key to this vault. Keep it safe.
